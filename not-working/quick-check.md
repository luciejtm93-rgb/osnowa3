# Домен не открывается: быстрая проверка

Не паникуйте — сначала пройдите этот чеклист.

## Шаг 1. Проверьте сколько времени прошло

Если вы только что сменили NS или добавили записи — просто подождите. Изменения могут применяться до 24 часов.

→ [Когда домен начинает работать](../quick-start/when-domain-works.md)

## Шаг 2. Проверьте NS на dnschecker.org

Откройте [dnschecker.org](https://dnschecker.org), введите домен, посмотрите NS.

- Показывает новые NS → всё идёт правильно, ждите
- Показывает старые NS везде → NS не применились → [Почему NS не меняются](../setup/ns-not-changing.md)

## Шаг 3. Попробуйте в режиме инкогнито

Иногда виноват кеш браузера. Откройте домен в новой вкладке инкогнито или с другого устройства.

## Шаг 4. Проверьте статус домена в боте

Откройте бота → **Мои домены** → выберите домен. Посмотрите:
- Есть ли блокировки?
- Какие NS указаны?
- Не истёк ли срок домена?

## Шаг 5. Определите причину по симптому

| Симптом | Куда идти |
|---|---|
| Бот показал ошибку при настройке NS | [Почему NS не меняются](../setup/ns-not-changing.md) |
| NS сменились, но открывается старый сайт | [Почему открывается старый сайт](old-site-showing.md) |
| У других открывается, у меня нет | [Почему только у меня](works-for-others.md) |
| Домен заблокирован | [Client Hold / Server Hold](../blocks/client-server-hold.md) |
| Нет HTTPS | [Что такое 1REG DNS и когда лучше Cloudflare](../setup/1reg-dns-vs-cloudflare.md) |

{% hint style="warning" %}
Прошло больше 24 часов и ничего не помогло — напишите в поддержку. Укажите домен и скриншот из бота с текущими NS.
{% endhint %}
